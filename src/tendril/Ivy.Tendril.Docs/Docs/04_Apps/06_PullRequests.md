# Pull Requests

<Ingress>
The Pull Requests app centralizes your automated integration pipeline, allowing direct Github manipulation inside the Tendril Desktop.
</Ingress>

## Objective

Tendril is designed specifically to not mutate your existing working repositories forcefully. All execution happens on specialized branches. 

When you authorize a Review through the interface sequentially, Tendril fires a background Promptware explicitly assigned to read your changed `.diffs` and summarize them via an automated Github Payload via `gh`.

## Managing State natively

Instead of tabbing backwards to a web browser, the Pull Requests application tracks the state natively:

- **Pending / Open** — Represents an AI-generated implementation ready for upstream codebase synchronization.
- **Merged** — Closed out cleanly and recorded functionally onto the Project repository default stream.

If Github CI/CD actions fail remotely, human maintainers can diagnose issues, alter logic, and resubmit updates natively across the same execution pathways. You can directly **Merge** PRs generated by the AI from this interface to finalize the AI lifecycle smoothly.
