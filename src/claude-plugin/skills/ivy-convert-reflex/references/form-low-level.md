# Form (Low Level)

Reflex's low-level form API provides granular control over form structure, validation, and submission. It uses composable primitives (`form.root`, `form.field`, `form.control`, `form.label`, `form.message`, `form.submit`) to build forms piece by piece, with built-in client-side and server-side validation wired to accessibility-friendly error messages.

## Reflex

```python
rx.form.root(
    rx.form.field(
        rx.flex(
            rx.form.label("Email"),
            rx.form.control(
                rx.input(
                    placeholder="Email Address",
                    type="email",
                ),
                as_child=True,
            ),
            rx.form.message(
                "Please enter a valid email",
                match="typeMismatch",
            ),
            rx.form.submit(
                rx.button("Submit"),
                as_child=True,
            ),
            direction="column",
            spacing="2",
            align="stretch",
        ),
        name="email",
    ),
    on_submit=lambda form_data: rx.window_alert(form_data.to_string()),
    reset_on_submit=True,
)
```

## Ivy

Ivy does not have a low-level composable form primitive API. Instead, it uses a model-driven `ToForm()` approach that auto-generates fields from a C# record/class. Validation is handled via DataAnnotations and programmatic `.Validate()` calls. The `Field` widget wraps individual inputs with labels, descriptions, and required indicators.

```csharp
public class EmailFormExample : ViewBase
{
    public record EmailModel
    {
        [Required(ErrorMessage = "Email is required")]
        [EmailAddress(ErrorMessage = "Please enter a valid email")]
        public string Email { get; set; } = "";
    }

    public override object? Build()
    {
        var model = UseState(() => new EmailModel());
        var client = UseService<IClientProvider>();

        UseEffect(() =>
        {
            if (!string.IsNullOrEmpty(model.Value.Email))
            {
                client.Toast($"Submitted: {model.Value.Email}");
            }
        }, model);

        return model.ToForm("Submit")
            .Label(m => m.Email, "Email")
            .Required(m => m.Email);
    }
}
```

## Parameters

| Parameter | Reflex | Ivy |
|-----------|--------|-----|
| Form container | `rx.form.root(...)` | `model.ToForm()` |
| Field wrapper | `rx.form.field(..., name="key")` | `.WithField()` on an input, or auto-generated by `ToForm()` |
| Label | `rx.form.label("Email")` | `.Label(m => m.Email, "Email")` |
| Input control | `rx.form.control(rx.input(...), as_child=True)` | Auto-generated from model type, or `.Builder(m => m.Field, s => s.ToTextInput())` |
| Validation message | `rx.form.message("error text", match="typeMismatch")` | `[EmailAddress(ErrorMessage = "...")]` DataAnnotation or `.Validate<T>(...)` |
| Submit button | `rx.form.submit(rx.button("Submit"), as_child=True)` | Submit button text passed to `ToForm("Submit")` |
| Submit handler | `on_submit=handler` (receives `dict`) | `UseEffect` reacting to model state change, or `UseForm()` hook with `onSubmit` |
| Reset on submit | `reset_on_submit=True` | `model.Set(new Model(...))` to reset manually |
| Server-side validation | `server_invalid` on field + `force_match` on message | `.Validate<T>(m => m.Field, val => (bool, "message"))` |
| Client-side validation | `match="typeMismatch"` / `match="valueMissing"` etc. | Not supported (validation runs server-side via Blazor) |
| Composable primitives | `form.root > form.field > form.control/label/message` | Not supported (model-driven generation only; `UseForm()` for partial control) |
| Field description/help | Not supported | `.Description(m => m.Field, "...")` / `.Help(m => m.Field, "...")` |
| Field grouping | Not supported | `.Group("Group Name", m => m.A, m => m.B)` |
| Conditional visibility | Manual via `rx.cond(...)` | `.Visible(m => m.Field, m => m.SomeBool)` |
| Layout control | Manual via `rx.flex(...)` | `.Place(...)` / `.PlaceHorizontal(...)` |
| Dialog/Sheet form | Not supported | `.ToDialog(...)` / `.ToSheet(...)` |
| as_child composition | `as_child=True` to compose custom components | Not applicable (uses `.Builder()` for custom inputs) |
